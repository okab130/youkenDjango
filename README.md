# Django要件定義テンプレート集

Django開発プロジェクトのための包括的な要件定義テンプレート集です。業務システム開発において、要件定義フェーズで活用できる6つの業務ドメイン別テンプレートを提供します。

## 📋 収録テンプレート

### 1. SCM要件定義テンプレート
サプライチェーン管理システム全般をカバーする統合的なテンプレート
- 調達管理（発注・納期管理）
- 在庫管理（入出庫・ロット管理）
- 販売管理（受注・出荷管理）
- 物流管理・生産管理連携

### 2. 在庫管理システム要件定義テンプレート
倉庫・在庫管理に特化したテンプレート
- リアルタイム在庫管理
- ロケーション管理（倉庫・棚番）
- ロット・トレーサビリティ管理
- 棚卸機能（実地棚卸・循環棚卸）
- バーコード/RFID対応

### 3. 発注管理システム要件定義テンプレート
購買・発注業務に特化したテンプレート
- サプライヤー管理・評価
- 見積管理（RFQ・見積比較）
- 発注承認ワークフロー
- 納期管理・進捗追跡
- 請求書照合・支払管理

### 4. 製造管理システム要件定義テンプレート（金属加工業特化）
金属加工業の製造管理に特化したテンプレート
- 生産計画・MRP（資材所要量計画）
- 製造指示・実績管理
- BOM（部品表）管理
- 工程管理（切削・プレス・溶接等）
- 品質管理・検査管理
- 設備稼働管理・保全管理
- 原価管理（標準原価・実際原価）

### 5. 財務会計システム要件定義テンプレート
財務会計業務の全般をカバーするテンプレート
- 仕訳入力・承認ワークフロー
- 勘定科目管理（階層構造・補助科目）
- 売掛金・買掛金管理
- 固定資産管理・減価償却
- 予算管理・予実対比
- 月次決算・年次決算
- 財務諸表（P/L・B/S・C/F）
- 税務申告対応（法人税・消費税）

### 6. 配送管理システム要件定義テンプレート
物流・配送業務に特化したテンプレート
- 配送計画・配車管理
- ルート最適化（AI対応）
- ドライバーアプリ連携
- GPS追跡・リアルタイム配送状況
- 顧客向け追跡ページ
- 荷物管理・在庫管理
- 料金計算・請求管理

## 🎯 特徴

### ✅ 実践的な構成
- 実際の業務要件定義で即座に活用できる詳細な項目
- 曖昧性を排除するためのチェックリスト
- 業務フロー記述テンプレート

### ✅ Django開発に最適化
- Djangoモデル設計を意識したエンティティ定義
- Django標準機能（Admin、認証、ORM等）の活用を考慮
- RESTful API、Celery、Django Channels等の技術選択ガイド

### ✅ 非機能要件の充実
- 性能要件（レスポンスタイム、データ量見積）
- セキュリティ要件（認証・認可・暗号化）
- 法令対応（電子帳簿保存法、個人情報保護法等）

### ✅ 段階的開発対応
- MVP（必須機能）と将来対応機能の明確な分離
- Phase別の優先度設定
- リスク管理と対応策

## 📚 使用方法

### 1. テンプレートの選択
開発するシステムに最も近いテンプレートを選択してください。複数の業務ドメインにまたがる場合は、複数のテンプレートを参照・統合することも可能です。

### 2. プロジェクト固有の要件記入
`markdown
- [ ] 該当する項目にチェック ✓
- [例: 具体的な数値や内容] を自社の要件に書き換え
- 不明な項目は「TODO」と記載し、後で確認
- 該当しない項目は「N/A」と記載
`

### 3. AIによる要件確認（推奨）
記入完了後、GitHub Copilot等のAIに以下のような指示で要件確認を依頼:
`
「この要件定義を確認し、曖昧な点・矛盾・不足を指摘してください」
「この要件に基づいてDjangoモデル設計を開始してください」
`

### 4. 設計への移行
要件定義完了後、以下の順序で設計を進めます:
1. データモデル設計（models.py）
2. 機能設計（URLルーティング・ビュー設計）
3. UI設計（ワイヤーフレーム・画面レイアウト）
4. 外部連携設計（API仕様）

## 🔍 各テンプレートの詳細構成

各テンプレートは以下の14-15章で構成されています:

1. **プロジェクト基本情報** - システム概要、目的、範囲定義
2. **ビジネス要件** - 解決したい課題、ユーザーロール、業務フロー
3. **機能要件** - 必須機能（MVP）、将来対応機能
4. **非機能要件** - 性能、セキュリティ、可用性、法令対応
5. **データモデル要件** - 主要エンティティ、属性定義
6. **画面要件** - 必須画面リスト、画面優先度
7. **UI/UX要件** - デザイン方針、操作性要件
8. **外部連携要件** - システム間連携、API連携
9. **技術要件** - Django技術スタック、フロントエンド、認証
10. **制約・前提条件** - プロジェクト制約、技術的制約、業務的制約
11. **リスクと対応策** - リスク分析、対応策
12. **成功基準（KPI）** - 定量的指標、定性的指標
13. **移行計画** - データ移行、段階的移行計画、教育訓練
14. **業務特有の考慮事項** - 各業務ドメイン固有の要件
15. **次のステップ** - 設計フェーズへの移行ガイド

## 🛠️ 技術スタック（推奨構成）

各テンプレートでは以下の技術スタックを推奨しています:

### バックエンド
- **Django**: 4.2 LTS
- **Python**: 3.11+
- **データベース**: PostgreSQL
- **キャッシュ**: Redis
- **タスクキュー**: Celery + Redis
- **リアルタイム通信**: Django Channels（必要に応じて）

### フロントエンド
- **テンプレートエンジン**: Django Templates
- **CSSフレームワーク**: Bootstrap 5 / Tailwind CSS
- **JavaScript**: Vue.js / Alpine.js / React
- **PDF生成**: WeasyPrint / ReportLab
- **Excel生成**: openpyxl

### インフラ
- **Webサーバー**: Nginx + Gunicorn
- **デプロイ**: AWS / GCP / Azure / オンプレミス
- **CI/CD**: GitHub Actions / GitLab CI
- **監視**: Sentry / Datadog
